## s11 光斑V4重写（日照金山风格·Canvas渲染）
- **用户反馈**：V3光斑只能上下移动（X轴跟踪bug）+ 亮度太暗
- **根因分析**：
  - X轴bug：旧代码用CSS radial-gradient的`at Xpx Ypx`，但W变量在不同作用域可能被覆盖
  - 太暗：遮罩opacity 0.96 + 内层光晕太弱（0.06~0.10 alpha）
- **V4方案：Canvas渲染替代CSS gradient**
  - 单个`<canvas id="spotlight-canvas">` 替代3个div
  - 第1层：fillRect深色遮罩 → destination-out挖洞（RadialGradient alpha渐变，圆心全透明）
  - 第2层：`globalCompositeOperation: 'lighter'` 叠加暖金色光（中心rgba(255,220,130,0.22)）
  - 第3层：金色粒子（每3帧生成，lighter模式绘制，缓慢上飘衰减）
  - 中心强光点：rgba(255,250,220,0.35) 模拟太阳光心
- **坐标修复**：
  - 鼠标/触摸：用`e.clientX / window.innerWidth`（不依赖W变量，避免作用域问题）
  - 摄像头：`window._handCenterNX`直接用（不做1.0-反转），方向不对再调
  - 所有位置变量用归一化0-1（spotX/spotY/spotTX/spotTY），渲染时乘canvas尺寸
- **亮度提升**：遮罩从0.96降到0.88，光区中心alpha从0.10提到0.22~0.35
- **光斑半径**：spotR/spotR_target替代spotRadius，平滑过渡0.15系数

## s11 光斑V5：手势范围修复（灵敏度放大）
- **用户反馈**：手势只能照亮画面中间一小块区域（椭圆形），鼠标却能照全屏
- **根因**：`_handCenterNX = skinBlob中心 / 160`，手在摄像头画面中无法触及边缘 → 值域仅约0.15~0.85；而鼠标用`clientX/window.innerWidth`覆盖完整0~1
- **修复方案**：以0.5为中心放大偏移量——X轴×2.8、Y轴×2.4，clamp到[0,1]。手在摄像头范围内移动即可覆盖整个屏幕

## s11 光斑V5b：摄像头追踪不灵敏+范围仍不够
- **用户反馈**：手势还是不能照全屏，而且不灵敏，和鼠标效果差太远
- **根因**：
  1. renderSpotlight的0.25慢速平滑对摄像头输入造成双重延迟（摄像头本身15-30fps+平滑收敛8-10帧≈150ms额外延迟）
  2. 灵敏度2.8×/2.4×仍不够（手实际值域可能更窄如0.25-0.75）
  3. skinBlob.total阈值200偏高，丢帧
- **修复**：
  1. 新增spotCamX/spotCamY摄像头专用快速EMA坐标，0.65系数（vs 鼠标0.25）
  2. spotCamInputActive标记区分摄像头/鼠标模式，分别走不同平滑通道
  3. 灵敏度提升到3.5×/3.0×
  4. skinBlob阈值从200降到100
  5. 鼠标/触摸事件自动切回鼠标模式

## s11 光斑V6：触控板模式（彻底解决手势范围问题）
- **用户反馈**：V5b灵敏度放大3.5×/3.0×仍然只能照中间，边缘到不了
- **根因**：线性放大方案本质上有天花板——手在物理空间不可能移到摄像头画面最边缘，无论放大多少倍都有死角
- **全新方案：触控板/trackpad模式**
  - 不再用手在摄像头中的绝对位置映射到屏幕
  - 改用手的移动增量（delta）驱动光斑位置——手往左移，光斑往左走；手往右移，光斑往右走
  - 小移动累积成大位移，3.5×增量放大，保证任何角落都能到达
  - 类似笔记本触控板，和鼠标体验一致
- **技术细节**：
  - `spotPrevRawX/Y`记录上一帧手位置，每帧算dx/dy增量
  - `spotTX += dx * 3.5`，clamp到[0,1]
  - 手离开画面时重置prev为-1，避免回来时跳跃
  - 摄像头EMA平滑系数0.5（vs鼠标0.25）

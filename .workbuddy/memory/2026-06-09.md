## 昆仑谣 v3.2 挥手转场修复（6月10日）
- **用户反馈**："挥手转场这个交互一直没有反应"
- **根因**：`handPositions`追踪的是运动质心(motionCX)——所有运动像素的加权中心。挥手时像素分散在轨迹两侧，质心几乎不动，导致`dirX`永远<2，挥手要么被误判为出拳，要么完全不触发
- **修复1**：在肤色分析后，用**肤色区域中心**替换handPositions最后一条记录——肤色中心真正跟随手的位置
  ```js
  if(skinBlob.total>300 && handPositions.length>0){
    handPositions[handPositions.length-1]={
      x:(skinBlob.minX+skinBlob.maxX)/2/STEP,
      y:(skinBlob.minY+skinBlob.maxY)/2/STEP
    };
  }
  ```
- **修复2**：挥手判定阈值从`dirX>2`降到`dirX>1.2`，出拳的兜底阈值同步调整
- **影响**：挥手(open)手势现在能被正确检测，"切场景"功能(s1/s2/s3/s4)和其他场景专属交互均可正常触发

## 昆仑谣 v3.4 青鸟场景握拳爆墨修复（6月10日）
- **用户反馈**："青鸟那里握拳爆墨没有反应"
- **根因**：握拳是**静态姿势**——手指合拢后手几乎不动，帧差分运动能量(energyPeak)远低于2.5触发阈值，永远进不了手势判定分支。现有系统只能检测"动态运动"（挥手/出拳），无法检测"静态姿态"（握拳hold）
- **修复**：新增**握拳姿态检测**，和1指检测平行：
  - `fistPoseDetected`：skinBlob存在(>300) + 无手指探出(!oneFingerDetected) + blob紧凑(宽高比0.6~1.5、宽<60、高<60、面积<2500)
  - `_fistPoseStable`稳定计数器：连续10帧检测到→触发，3帧衰减
  - 触发效果按场景区分：青鸟→墨爆+星光、昆仑→台阶光纹、树→震动、瑶池→波涛、尾声→墨韵、其他→墨爆
  - 青鸟场景专门调用`punchEffect()`产生28颗墨粒子+涟漪，加12颗星光粒子
- **HUD增强**：✊ FISTPOSE指示 + FS:计数，皮肤框变红+✊标记

## 昆仑谣 v3.5 比1手势修复（6月10日）
- **用户反馈**："最后什么比1根本没反应"
- **根因（双重）**：
  1. **比1检测条件过于严苛**：`cnt>25`手掌过渡阈值太高、`fingerMax<18`太窄、`_fingerStable>=12`要求12帧连续稳定，在160×120低分辨率图像中几乎不可能同时满足
  2. **握拳检测抢走比1**：比1时手指检测闪烁不稳定，当`oneFingerDetected=false`的帧，握拳检测立即生效（blob看起来紧凑），`_fistPoseStable`先到10触发，消耗掉cooldown，比1永远等不到12帧
- **修复1**：放宽比1检测阈值——`skinBlob.total>200`（300→200）、blob高度>20（30→20）、手掌过渡`cnt>15`（25→15）、`fingerMax<22`（18→22）、比例`1.8`（2.2）、手指高度>6行（10→6）
- **修复2**：握拳检测排除瘦高型blob（`aspect>=0.7`，原来>0.6），比1时手指朝上blob高>宽，不再被误判为握拳
- **修复3**：比1稳定帧数从12降到8，衰减从-2降到-1，更容易达到触发阈值

## 昆仑谣 v3.6 手势灵敏度全面优化（6月10日）
- **用户反馈**："交互手势为什么不灵敏"
- **根因**：6个瓶颈叠加，导致手势检测极度迟钝：
  1. **采样率低**：`_pfSkip<3`，每3帧才处理1次→60fps实际只有~10fps，手挥完了还没采样到
  2. **能量阈值高**：`energyThreshold=2.5`，小幅运动能量不够
  3. **冷却太长**：`GESTURE_CD=40帧`≈4秒，触发一次后很久不能再触发
  4. **帧差阈值高**：`diff>12`，轻微运动被忽略
  5. **运动像素门槛高**：`motionCount>10`，运动面积太少不算
  6. **峰值衰减快**：`peakDecay=0.92`，脉冲还没累积就被衰减掉了
- **修复**：
  | 参数 | 旧值 | 新值 | 说明 |
  |------|------|------|------|
  | _pfSkip | 3 | 2 | 采样率翻倍，~10fps→~15fps |
  | energyThreshold | 2.5 | 1.5 | 能量门槛降低40% |
  | GESTURE_CD | 40 | 20 | 冷却时间减半（4s→2s） |
  | diff | >12 | >8 | 帧差阈值降低，捕捉更细微运动 |
  | motionCount | >10 | >5 | 运动像素门槛减半 |
  | peakDecay | 0.92 | 0.96 | 衰减更慢，脉冲更容易累积触发 |
  | 举手cooldown | CD*3 | CD*2 | 跟随主CD调整 |
